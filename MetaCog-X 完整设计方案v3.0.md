# MetaCog-X 完整设计方案（v3.0）

## 1. 概述与设计哲学

### 1.1 目标
构建一个**低能耗、高自适应**的神经网络架构，使模型具备：
- **困境条件化的元认知**：仅在认知困境（高不确定性、重复循环、信息匮乏）时启动深度监控与调控；
- **默认模式自我感**：一个持续运行的轻量级循环网络提供稳定的“自我”背景，支持直觉、好奇心和长期一致性；
- **三级分层调度**：L1反射检测 → L2战术调控（含在线学习与工具调用）→ L3战略复盘。

### 1.2 设计原则
- **稀疏激活**：元认知模块默认休眠，仅当困境门控触发时投入额外计算。
- **低增量开销**：所有额外模块的参数量和FLOPs增量控制在主模型的10%以内。
- **兼容性**：可基于现有Transformer（如GPT、LLaMA）微调，无需改变预训练权重结构。

## 3. 模块详细设计

### 3.1 默认模式网络 (DMN) —— 持续自我感背景

**目的**：提供一个持续运行的、低功耗的循环神经网络，模拟人脑默认模式网络的功能，产生稳定的“自我向量”和“意外信号”。

- **结构**：单层GRU，隐藏状态维度 `d_self = 16`。
- **输入特征**（每处理一个token采集一次，或每隔K个token采样）：
  - 最近N步的注意力熵均值、方差（来自标准或认知模式）
  - 当前步logits的最大概率、熵
  - 觉知池统计量（若激活）
  - 前一层隐藏状态的全局均值
- **更新规则**：
  - 无外部输入时（如模型空闲），GRU仍以时间步迭代，输入设为0向量，并乘以衰减因子。
  - 有外部输入时，正常更新。
- **输出**：
  - **自我向量** `h_self`：GRU的隐藏状态，作为背景偏置。
  - **意外信号** `surprise`：一个小型MLP接收 `h_self` 和当前特征，输出标量0~1，表示输入与预期模式的差异。
- **注入方式**：
  - 在标准模式每层注意力logits中添加 `h_self` 的线性投影偏置。
  - 在元认知模式中，将 `h_self` 与认知粒子的 `awareness` 拼接。
  - `surprise` 作为L1门控的额外输入特征。

### 3.2 L1 困境门控 (Dilemma Gate)

- **输入**：
  - 来自Transformer副产品的特征：各层注意力熵平均值、logits最大概率、logits熵、当前序列长度与预期长度比值、重复计数。
  - 来自DMN的 `surprise` 信号。
- **网络**：2层MLP（输入维度 = 层数 + 3 + 1 + surprise），输出标量 `dilemma_score`（Sigmoid）。
- **触发策略**：连续2步 `dilemma_score > 0.7` 则激活元认知模式；退出需连续3步 `<0.3`。
- **训练**：使用人工标注的“困难步骤”作为正样本，正常步骤为负样本，二元交叉熵损失。

### 3.3 认知粒子生成器 (Particle Generator)

仅在进入元认知模式时运行。将当前隐藏状态 `x` 转换为三元组：
```
content, meta, awareness = split(linear(x), [d_model, d_meta, d_aware])
```
- `content`：保留原始语义，与主模型兼容。
- `meta`：维度 `d_meta=32`，前4维经过约束（置信度0-1、深度非负、策略softmax、熵非负），其余可学习。
- `awareness`：维度 `d_aware=16`，初始化为0。

### 3.4 认知Transformer块（元认知模式层）

每一层替换标准注意力为**三重注意力**，并添加稀疏控制器信号。

#### 3.4.1 三重注意力 (Triple Attention)
采用**加性偏置**方式融合meta和awareness，避免额外Q/K矩阵：

```python
# content QKV
Q = linear_q(content); K = linear_k(content); V = linear_v(content)
# 标准注意力logits
attn_logits = (Q @ K.transpose(-2,-1)) / sqrt(d_head)
# meta偏置
meta_bias = linear_meta_bias(meta).view(B, L, H)  # [B,L,H]
meta_bias = meta_bias.unsqueeze(2) - meta_bias.unsqueeze(1)  # [B,H,L,L]
# awareness偏置
aware_bias = linear_aware_bias(awareness).view(B, L, H).unsqueeze(2) * 0.1
attn_logits = attn_logits + meta_bias + aware_bias
# 掩码、softmax、输出
```

#### 3.4.2 稀疏元认知控制器
位于每层，输入当前层的 `meta` 全局平均、觉知池统计量、DMN自我向量、层索引，输出三个标量：
- `temp_factor ∈ [0.8,1.2]`：调节注意力softmax温度。
- `skip_prob ∈ [0,1]`：跳过本层计算的概率（训练时随机，推理时阈值化）。
- `mem_strength ∈ [0.5,1.0]`：调节觉知池读取权重。

网络结构：嵌入层索引 + MLP。

#### 3.4.3 觉知池 (Awareness Pool)
- 容量 `C=32`，存储历史 `awareness` 的全局均值（每层每步一个）。
- 在线计算均值、标准差、趋势、熵。
- 为元认知控制器提供统计特征，同时为DMN提供输入。

### 3.5 L2 战术调度器 (Tactical Scheduler)

**功能**：根据当前困境的时序特征，从**元认知策略库**中检索最优干预策略，或触发**工具调用**。

> **[v3.1 修订]** 采纳"系统优化建议 v3.1"P0-2：彻底删除**主模型参数在线更新**逻辑（含EWC等任何参数微调机制），改用**策略库检索**替代。原因：参数在线更新不可避免地导致灾难性遗忘（无论EWC怎么调），且在线梯度更新延迟高、开销大。策略库是灰箱可解释方案。

- **输入**：最近 `T=10` 步的特征序列（dilemma_score、logits熵、模型置信度、DMN surprise、干预效果等），维度 `d_seq`。
- **编码器**：单层 LSTM → 最后隐藏状态。
- **策略库**（内存字典，key=困境类型哈希，value={strategy_id, confidence, expected_improvement, cost}）：
  - 预置策略：`调温度0.95`、`调温度1.05`、`触发重置（清空觉知池+回退N步）`、`调用外部工具`、`提示词重写`。
  - 运行时由 L3 复盘直接写入/更新，**绝不修改主模型参数**。
- **输出头**：
  - `strategy_id`（已注册策略之一）+ `confidence`
  - `call_tool`（二分类，并输出 tool_id，仅当策略是"调用工具"时为 true）
- **训练**：PPO 训练策略检索头（只训练策略选择，不训练参数更新），奖励函数：
  ```
  R = task_success - α*tool_call_cost - β*intervention_count
  ```
  （**无 online_learning_cost 项**，因为已无参数在线更新）
- **自我效能预测辅助任务**：从 LSTM 状态额外预测未来 K 步的任务成功率，MSE 损失，帮助调度器学会前瞻性判断。

### 3.6 开悟触发器 (Enlightenment Trigger)

嵌入在L2调度器中，当满足以下任一条件时触发“框架切换”：
- 连续多步 `dilemma_score` 高但元认知干预无效。
- 重复次数极高（>3个相同token）。
- 过度自信且任务失败（高置信度但最终结果错误）。

触发的动作：
- **内部重置**：清空觉知池、重置DMN短期记忆、回退若干token重新生成。
- **工具调用**：通过`call_tool`动作请求外部知识或计算。

### 3.7 L3 战略性复盘 (Strategic Reflection)

- **触发**：任务完全结束后（成功或失败），且该任务被标记为"重要"（如用户反馈差、能耗高）。
- **输入**：整个任务期间的认知日志（L1分数、控制器信号、觉知池快照、调度器决策、最终结果）。
- **编码器**：一个小型 Transformer（或预训练主模型的浅层），输出经验摘要向量。
- **用途**：
  - **[v3.1 修订]** **直接写入元认知策略库**（困境特征 → 干预策略 → 效果评分三元组），**绝不修改主模型任何参数**。
  - 生成改进规则（提示词模板、超参数调整）存入策略库。
- **[v3.1 新增] 灰度A/B机制**：新策略先在 10% 流量上验证，效果达标（干预后成功率提升 ≥ 5%）才全量。
- **[v3.1 新增] 结构化复盘输出**：优先生成自然语言的错误分析、经验总结、改进建议（可由小模型生成），再转化为策略库条目。
- **训练**：元学习评估"新策略入库后下一批任务成功率变化"作为奖励（评估指标：策略采纳率、错误重复率、任务成功率变化）。

## 4. 数据流与协同示例

1. 模型处理简单任务（如日常对话）：L1门控输出低分 → 标准模式运行 → DMN持续更新自我向量但无干预 → 无额外开销。
2. 遇到复杂数学题：L1检测到高熵、重复 → 连续2步超过阈值 → 切换元认知模式 → 生成认知粒子，激活三重注意力，觉知池开始记录。
3. 元认知模式中，稀疏控制器调节温度、跳过率，帮助模型聚焦；觉知池统计量送入L2调度器。
4. L2调度器根据时序特征判断“正在接近死循环”，触发工具调用（计算器）→ 获取结果注入上下文 → 继续推理。
5. 任务成功 → L3复盘模块提取经验（“遇到复杂计算时提前调用工具”）→ 存入策略库。
6. 若任务反复失败且高置信度，触发开悟：内部重置，换一种解题思路。
7. 困境解除后，连续低分 → 退出元认知模式，回归标准模式，节省能耗。

## 5. 训练策略

### 5.1 预训练（无监督，标准模式 + DMN初始化）
- 主任务：语言建模（交叉熵）。
- 辅助损失：
  - 时序一致性：相邻步meta向量MSE（仅困境步骤）。
  - 自我预测：用 `h_self` 预测未来几步的awareness统计量。
  - surprise与困境的相关性：鼓励surprise高时dilemma_score也高。
- DMN GRU与主模型一起训练。

### 5.2 元认知微调（强化学习，冻结主模型参数）
- 任务集：多步推理、数学题、需要工具的任务。
- 可训练模块：L1门控、元认知控制器、L2调度器**的策略检索头**、DMN的投影层。
- **[v3.1 修订]** 奖励函数：`R = success - λ1*energy_ratio - λ2*intervention_count - λ3*tool_cost`（删除 `online_learning_cost` 项，因为不再有参数在线更新）
- **[v3.1 修订]** 策略库为 dict 查找表，初始人工预置 + L3 复盘运行时写入。
- 使用 PPO，困难样本优先采样。

### 5.3 开悟与复盘训练
- 构造对抗任务（初始条件错误、无解问题），通过模仿学习让模型学会重置和工具调用。
- L3复盘模块单独训练：将任务日志映射到改进规则，用元学习评估规则的有效性。

## 6. 评估指标

| 指标 | 定义 | 目标 |
|------|------|------|
| **困境检测F1** | L1对真实困境步骤的查准/查全 | >0.8 |
| **非困境能耗比** | 简单任务上总FLOPs vs 基线FLOPs | <1.05 |
| **困境能耗节省** | 复杂任务上总FLOPs降低 | >20% |
| **干预有效率** | 元认知干预后任务成功率提升 | +15% |
| **开悟解脱率** | 陷入死循环后成功跳出并解决 | >50% |
| **调度器准确率** | 在线学习/工具调用决策合理 | >70% |
| **复盘收益** | 复盘后在新任务上成功率提升 | +5% |

## 7. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| L1误触发频繁切换 | 滞后机制、代价敏感训练；增加切换惩罚 |
| 元认知模式开销过大 | 三重注意力采用加性偏置，可与FlashAttention融合；skip_prob动态调整 |
| 控制器在困境模式坍缩 | 只困境激活，保证训练样本有意义；添加熵正则 |
| **[v3.1 删除]** 在线学习灾难性遗忘 | **[v3.1 解决]** 彻底删除主模型参数在线更新（包括EWC），改用策略库检索，灰箱可解释 |
| DMN自我向量退化 | 强制时序平滑正则，并确保surprise与困境相关性 |
| **[v3.1 新增]** 策略库覆盖不足 | L3复盘持续写入新策略；预置策略先覆盖 3-5 个常见困境类型 |
| **[v3.1 新增]** 策略误匹配 | 策略库带 confidence 评分；低置信度时回退到"调默认温度"安全策略 |

## 8. 实现路线图（面向工程）

1. **基础骨架**：实现标准模式+元认知模式切换逻辑，添加L1门控（随机初始化）。
2. **核心模块**：认知粒子生成器、三重注意力、觉知池、稀疏控制器（用模拟控制信号替代）。
3. **训练数据**：在数学推理数据集上人工标注困境步骤，训练L1门控。
4. **RL环境**：搭建多步推理任务环境，使用PPO训练控制器和调度器。
5. **工具集成**：接入计算器、搜索API，完善开悟触发器。
6. **复盘模块**：离线实现经验库和元学习。
7. **联合调优**：在Big-Bench Hard、GSM8K、MATH等基准上评估。
