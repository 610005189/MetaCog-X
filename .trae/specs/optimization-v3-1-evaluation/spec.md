# MetaCog-X 系统优化建议 v3.1 — 有效性评估与后续路线

## Overview
- **Summary**: 对用户更新后的《系统优化建议3.1.md》进行逐条有效性/可行性评估，与已实现代码（L1门控、模式切换、窄幅Controller）和v3.0完整设计方案对齐，给出采纳/延后/否决决策，产出可执行的下一阶段工作清单。
- **Purpose**: 解决"哪些建议真能落地、哪些是空中楼阁、先做什么"的问题，避免在不可行方向上浪费工期。
- **Target Users**: MetaCog-X 开发者（当前即本人）。

## 关键发现（前置结论）
更新后的建议3.1**不再是无关模板**——已针对性重写为 v3.0 架构的落地建议。但：
1. **所有建议中，真正当前可落地的只有 P2-5（三重注意力融合消融）和 P0-1 自监督预训练的思路**；
2. 多数建议（L2策略库、L3复盘闭环、DMN好奇心、跨模型迁移）针对尚未实现的模块，属于**设计输入而非当前编码任务**；
3. **最大遗漏**：跳过的 Task 5 A/B 训练验证是验证"条件激活是否真的解决Controller坍缩"的唯一核心实验，**必须先做**，否则后续工作建立在未验证假设上。

## 逐条建议评估

### P0-1 L1 困境门控增强
| 子建议 | 评估 | 决策 |
|---|---|---|
| 自监督预训练（N步预测误差自动标注） | 技术上可行但**当前 tiny byte-level 模型没有"预测误差"信号**（只有交叉熵 ppl）。v3.0已规定"人工标注+BCE"作为训练方式。自监督应留到多步推理数据集（GSM8K等）阶段 | **延后**，作为 L1 v2.0 设计输入 |
| 语义一致性 / 逻辑一致性特征 | 需要 embedding 语义空间对齐，tiny 模型无语义能力；逻辑一致性需要知识库，完全不可行 | **否决**，超出当前原型能力 |
| 自适应阈值（任务/负载动态调） | 纯工程开关，可做，但当前固定阈值 0.3/0.7 + 滞后已够用 | **延后**，等多任务场景再做 |

**整体结论**：P0-1 部分有价值（自监督预训练方向正确），但当前阶段不可执行。L1 现有 BCE 训练（F1=0.995）已足够好。

### P0-2 彻底替换参数式在线学习 → 策略库
| 子建议 | 评估 | 决策 |
|---|---|---|
| 完全删除参数在线更新 | v3.0风险表已标记 EWC 为缓解方案。**用策略库替代**是更优架构，彻底避免灾难性遗忘 | **采纳为L2核心设计原则** |
| 策略库三元组（特征→策略→评分） | 纯架构设计，不编码但决定 L2 实现方式 | **采纳** |
| L3 复盘写策略库而非改参数 | 同上 | **采纳** |

**整体结论**：P0-2 是**整个建议中最有价值的架构决策**，必须采纳为 v3.1 设计修订。

### P1-3 开悟触发器升级（框架一致性+多假设）
需要外部知识/工具接口，当前无工具模块。**延后**，等L2+工具调用实现后再设计。

### P1-4 L3 复盘闭环（结构化总结+策略转化管道+灰度A/B）
L3 本身都未实现。**延后**，设计输入。

### P2-5 元认知干预有效性验证
| 子建议 | 评估 | 决策 |
|---|---|---|
| 三重注意力融合方式对比（加性/乘性/拼接） | **完全可执行**，代码已实现加性偏置，加2个消融版本即可 | **立即执行**（Task C） |
| Controller增加注意力头掩码+推理步数限制 | 可行，但当前Controller才刚缩窄范围，先跑消融再扩展 | **延后** |
| 觉知池升级为时序滑窗统计 | 可行但优先级低 | **延后** |

**整体结论**：P2-5 是**当前最有价值的可执行实验**。

### P2-6 L2调度器灵活性
L2尚未实现。**延后**，设计输入。

### P3-7 DMN深化（好奇心、疲劳监控、自我模型）
v3.0设计了DMN但尚未实现。**设计输入**，实现DMN时吸收。

### P3-8 跨模型迁移
需要多个大模型，当前完全不可行。**否决**，v4.0再考虑。

## Goals
- G1: 完成 Task 5 A/B 训练验证，确认条件激活架构确实解决 Controller 坍缩
- G2: 执行三重注意力融合消融实验
- G3: 实现 DMN 默认模式网络（v3.0 下一个未实现核心模块），吸收 P0-1 的 surprise→L1 连接和 P3-7 好奇心驱动
- G4: 按"策略库替代参数在线学习"设计原则实现 L2 战术调度器骨架
- G5: 更新 v3.0 设计方案，标记采纳/修订点

## Non-Goals
- 不在 tiny byte-level 模型上实现语义一致性/逻辑一致性（无语义能力）
- 不实现自适应阈值（固定阈值+滞后够用）
- 不实现跨模型迁移（v4.0问题）
- 不在当前原型阶段做 L3 复盘闭环（依赖 L2+工具）

## Background & Context
当前实现覆盖 v3.0 模块：L1困境门控 ✓、模式切换 ✓、三重注意力 ✓、窄幅Controller ✓、觉知池 ✓、RL骨架 ✓。未实现：DMN、L2调度器、开悟触发器升级、L3复盘、工具接口。

## Functional Requirements
- **FR-1**: Task 5 A/B 训练脚本必须跑完 2000 step，输出 3 组 ppl + controller std + mode 切换统计
- **FR-2**: 三重注意力必须支持加性偏置（当前）、乘性门控、注意力拼接三种融合方式，ablation 脚本并排对比
- **FR-3**: DMN 实现单层 GRU(d_self=16)，提供 h_self 和 surprise 两个输出，surprise 接入 L1 作为额外输入特征
- **FR-4**: L2 调度器骨架基于"策略库检索"而非参数更新

## Non-Functional Requirements
- **NFR-1**: 所有新增模块参数量增量 ≤ 主模型 10%
- **NFR-2**: DMN 默认模式开销 ≤ 1% 额外 FLOPs

## Constraints
- 当前模型 tiny byte-level（d_model=128, n_layer=2, n_head=2, vocab=256），算力仅 CPU
- 不能引入新的外部依赖
- 所有实验可在 30 分钟内出结果

## Assumptions
- Task 5 A/B 在 CPU 上可在合理时间完成（≤60分钟）
- 三重注意力三种融合方式在 tiny 模型上差异可测量

## Acceptance Criteria

### AC-1: A/B 训练验证通过
- **Given**: runs/run_ab_v2.py 脚本
- **When**: 跑完 2000 step，每 500 step 输出 val ppl + controller std + mode 切换
- **Then**: metacog_conditional 组 controller temp_factor std > 0.05（不坍缩），ppl ≤ metacog_alwayson 组
- **Verification**: `programmatic`

### AC-2: 三重注意力消融完成
- **Given**: 同一 tiny 模型 + 同一数据 + 同一超参
- **When**: 分别跑加性偏置 / 乘性门控 / 注意力拼接 三种 TripleAttention
- **Then**: 输出 ppl + mode切换次数 + controller std 三组对比表
- **Verification**: `programmatic`

### AC-3: DMN 实现
- **Given**: DMN 模块集成进 metacogx_model.py
- **When**: forward 时 DMN 每步更新 GRU 隐藏状态，输出 h_self + surprise
- **Then**: h_self 维度 16，surprise ∈ [0,1]，surprise 作为 L1 额外输入特征
- **Verification**: `programmatic`

### AC-4: L2 策略库骨架
- **Given**: L2 调度器新建
- **When**: 检测到困境序列
- **Then**: 从策略库（字典）检索干预策略，输出 strategy_id + confidence
- **Verification**: `programmatic`

### AC-5: 设计方案更新
- **Given**: MetaCog-X 完整设计方案v3.0.md
- **When**: 标注采纳 P0-2（策略库替代参数在线学习）、延后项、否决项
- **Then**: 文档中 v3.1 修订点清晰可查
- **Verification**: `human-judgment`
